# GitHub workflow

- `main` – stabilní zdrojový kód
- `feature/...` – jednotlivé funkční změny
- tagy – `v1.4.0`, `v1.4.1`, ...
- Releases – později lze přikládat XLAM, EXE a SHA-256

Pro firemní nástroj doporučujeme privátní repozitář. Do repozitáře neukládejte certifikáty, privátní klíče, tokeny ani hesla.
